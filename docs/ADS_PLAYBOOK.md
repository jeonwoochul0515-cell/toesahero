# 파워링크 광고 운영 플레이북 — 심사 통과 후 실행 가이드

> 작성: 2026-07-20. 비즈채널 "퇴사히어로 웹사이트"(`bsn-a001-00-000000014578947`)가 심사 중(`UNDER_REVIEW`)인 동안 준비한 문서.
> 심사가 끝나 채널 상태가 `ELIGIBLE`이 되면 이 문서 순서대로 실행한다.

## 현재 등록 상태 (2026-07-20 API 실측)

- 캠페인: `퇴사히어로_최고신뢰` (`cmp-a001-01-000000010876508`, ELIGIBLE)
- 광고그룹: `최고신뢰8개` (`grp-a001-01-000000070405527`) 1개 — 채널 심사 대기로 PAUSED
- 키워드 20개 전부 이 한 그룹에 등록, 전부 `UNDER_REVIEW`
- 소재 1개: "직장 갈등, 변호사 상담 / 권고사직, 괴롭힘, 임금체불 문제를 변호사가 직접 대응합니다. 카톡 상담 가능" → 랜딩 `https://toesahero.com/` (홈)

**문제**: 검색 의도가 전혀 다른 20개 키워드("권고사직서양식" 같은 서류 찾기 vs "임금체불전문변호사" 같은 의뢰 의도)가 모두 홈 하나로 떨어진다. 의도 불일치 랜딩은 품질지수↓ → 클릭당 단가↑ → 전환율↓ 삼중 손해.

## 1단계 — 심사 통과 확인 (실행 전 필수)

```
GET /ncc/channels → 대상 채널 status == "ELIGIBLE" 확인
```

`UNDER_REVIEW`인 동안 재시도해도 항상 1018 에러만 반복된다 (CLAUDE.md 검색광고 API 주의사항 참조). 심사는 보통 영업일 1~3일.

## 2단계 — 키워드 그룹별 랜딩 매핑 (의도 정렬)

| 그룹 | 키워드 | 랜딩 | 근거 |
|---|---|---|---|
| A. 권고사직-서류 | 권고사직서양식, 권고사직서, 권고사직사직서 | `/resignation-letter` | 권고사직 확인서 양식 3종 제공 페이지. 서류 찾는 의도에 정확 일치 |
| B. 권고사직-상황 | 권고사직, 권고사직위로금, 권고사직회사불이익 | `/blog/recommended-resignation-severance-pay` (발행 후) 또는 홈 | 위로금·불이익 정보 의도. 블로그 발행 전이면 홈 유지 |
| C. 괴롭힘 | 직장내괴롭힘증거, 직장내괴롭힘처벌, 직장내괴롭힘노무사, 직장내괴롭힘사례, 직장괴롭힘 | `/harassment` | 전용 페이지 존재 |
| D. 해고 | 해고통지서, 해고예고통지서 | `/blog/dismissal-notice-requirements` (발행 후) | 현재 대응 콘텐츠 없음 → 블로그 발행 후 연결 |
| E. 임금체불 | 임금체불전문변호사, 체불임금확인서 | `/calc` (계산기) 또는 홈 | 임금체불전문변호사는 의뢰 의도 → 홈, 체불임금확인서는 정보 의도 → 블로그 발행 후 교체 |
| F. 절차·제도 | 노동청신고방법, 국선노무사, 중앙노동위원회재심 | 해당 블로그 글 | 재심은 기발행(`/blog/workplace-harassment-labor-office-appeal`), 나머지는 큐에 추가됨 |
| G. 브랜드 | 퇴사대행, 퇴사히어로 | 홈 | 브랜드·핵심 서비스 의도 |

**연계 준비 완료**: D·E·F 그룹용 블로그 주제 6개를 `content/blog/topics.json` 큐 최상단에 추가함(2026-07-20). 하루 1편 자동 발행이므로 약 6일 내 전부 발행 예정 — 심사 기간과 겹쳐 광고 시작 시점엔 랜딩이 준비돼 있을 것.

## 3단계 — 광고그룹 분리 (사용자 예산 확정 후)

- 그룹을 한 번에 다 쪼개지 말고, 우선 **A(서류)·C(괴롭힘)·G(브랜드)** 3개만 분리 신설하고 나머지는 기존 그룹 유지가 안전. 소재 재심사가 그룹 단위로 걸리기 때문.
- API 절차 (캠페인은 기존 `cmp-a001-01-000000010876508` 재사용, 신규 생성 금지 — 1018 에러):
  1. `POST /ncc/adgroups` — `nccCampaignId` + `pcChannelId`/`mobileChannelId`(= `bsn-...-14578947`) 필수
  2. `POST /ncc/keywords` — **각 키워드에 `"useGroupBidAmt": false` 필수** (누락 시 3916 에러)
  3. 기존 그룹에서 이동할 키워드는 `DELETE /ncc/keywords/{id}` — 빈 응답(204)은 정상 삭제이니 JSONDecodeError를 실패로 취급하지 말 것
  4. `POST /ncc/ads` — 그룹별 소재 등록 (아래 초안)
- 입찰가·일예산은 **사용자 확정 필요** — 임의 집행 금지 (기존 결정: 월 10~30만원 파일럿, 2~4주).

## 4단계 — 그룹별 소재 카피 초안 (변협 심의 준수: 과장·단정·승률 표현 없음)

- A(서류): "권고사직 확인서 양식 무료 제공 / 사직서·권고사직 확인서 양식을 무료로 받고, 서명 전 주의사항까지 확인하세요"
- C(괴롭힘): "직장 내 괴롭힘, 변호사와 상담 / 증거 정리부터 신고 절차까지, 변호사가 처음부터 함께 정리해 드립니다. 카톡 상담 가능"
- G(브랜드): 기존 소재 유지 ("직장 갈등, 변호사 상담 / 권고사직, 괴롭힘, 임금체불 문제를 변호사가 직접 대응합니다. 카톡 상담 가능")

> 타깃 고객은 갈등 회피형(사장과 직접 대화가 싫은 사람) — 카피는 "싸워드립니다"가 아니라 "대신·함께 정리해 드립니다" 톤 유지.

## 5단계 — 측정

- 광고 시작 후 2주: 네이버 광고관리시스템 보고서에서 키워드별 노출·클릭·전환(상담 신청) 확인.
- 상담 리드 출처 구분: Firestore `consultations`의 `meta.tool`/`source` 필드로 유입 도구 식별 가능. 광고 전용 UTM이 필요하면 랜딩 URL에 `?utm_source=naver_sa` 부착 검토 (사이트 코드 수정 불필요, 그대로 동작).

## 유사문서 주의 (큐 정리 참고)
