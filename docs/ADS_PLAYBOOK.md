## 미해결: 전환 추적 꺼져 있음

캠페인 `trackingMode: TRACKING_DISABLED`. 어느 키워드가 실제 상담 신청으로 이어졌는지 알 수 없어, 2주 뒤 성과 판단의 근거가 "클릭 수"까지밖에 안 된다.

해결하려면 네이버 **프리미엄 로그분석**(무료) 신청 후 발급되는 스크립트를 사이트에 심어야 한다 — 광고주센터 → 도구 → 프리미엄 로그분석에서 신청. 스크립트를 받으면 `index.html`에 삽입하는 작업은 코드로 처리 가능.

대안(즉시 가능): 랜딩 URL에 `?utm_source=naver_sa&utm_medium=cpc&utm_campaign=<키워드>`를 붙이면 유입 경로를 구분할 수 있다. **단 URL 변경은 재심사 대상**이라 지금은 보류(사용자 결정).

## 측정 — 2주 뒤 확인할 것

성과 조회는 API로 가능(`GET /stats?ids=<광고그룹ID>&fields=["impCnt","clkCnt","salesAmt","ctr","cpc"]&timeRange={"since":"...","until":"..."}`). 집계는 매시 정각 기준이라 당일 데이터는 늦게 반영된다.

판단 기준.
1. **노출이 나오는가** — 70원 키워드들이 실제로 노출되는지. 0이면 예산이 아니라 입찰가 문제이니 5위 시세를 다시 조회할 것.
2. **어떤 키워드가 클릭되는가** — 클릭이 특정 키워드에 쏠리면 그 키워드만 남기고 나머지를 정리해 예산 집중.
3. **하루 5,000원을 다 쓰는가** — 못 쓰고 남으면 입찰가를 올릴 여지가 있다는 뜻. 매일 소진되면 예산 증액을 검토.
4. **상담 신청이 늘었는가** — Firestore `consultations` 문서 수를 광고 전후로 비교. 전환 추적이 꺼져 있어 키워드 단위 연결은 아직 불가.
