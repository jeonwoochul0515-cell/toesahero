# 퇴사히어로 — 기능 고도화 실행계획 (일본 레퍼런스 기반)

**작성일**: 2026-06-18
**대상**: toesahero.com (Vite SPA + Cloudflare Pages Functions + Firestore + Toss + 카카오 + Claude)
**기반**: 일본 退職代行 시장 리서치(2026-06), `docs/ROADMAP.md`(전략·분기), `docs/research/02_japan_market.md`
**성격**: 전략 로드맵을 **빌드 가능한 기능 단위로 분해**한 실행계획. 각 기능에 구현 명세·수용기준·규정 체크를 붙였다.

## 0. 컴파스 — 무엇을 만들고, 무엇을 안 만드는가

일본 시장(유일한 성숙 시장, 100+ 업체)은 **일반업체 / 노동조합 / 변호사** 3계층으로 갈리고, 사람들이 **변호사 타입에 2~3배를 내는 이유는 정확히 미지급금 회수·손해배상 방어·분쟁**이다. 한국은 노조 우회 모델이 안 통해 **변호사 운영이 사실상 유일한 "교섭형"**.

→ **고도화 원칙: 싼 업체와 속도·가격 경쟁 금지. "변호사라서 되는 것"을 또렷한 상품·경험으로 만든다.**

### ⛔ 일본은 쓰지만 우리는 금지 (변협 「변호사 광고에 관한 규정」)
- "전액 환불 보장", "퇴직 성공률 100%", "최고·유일·업계 1위", 타 업체 비교
- 결과·금액 단정("○○만원 받습니다")
- → 대체 신뢰지표: **실적 건수, 사실 기반 후기, 법조문 근거** (현재 톤 유지)

## 2. P0 — 전환율 즉효 (결제 라이브 직후 최우선)

### P0-1. 손해배상 방어 상품화
- **목표**: "회사가 손배·위약금으로 협박해도 변호사가 직접 막는다"를 명시 상품/메시지로. (일본 변호사 타입의 핵심 수요.)
- **범위**: 홈·가격 섹션에 손배 방어 카피 추가 / 챗봇 시스템 프롬프트에 "손배 위협 시 변호사 대응" 분기 / 상담 인테이크에 `damageThreat: boolean` 플래그.
- **구현**:
  - `src/components/Pricing.tsx`·`Lawyer.tsx`에 "손해배상 위협 대응" 항목.
  - `functions/api/chat.ts` SYSTEM_PROMPT에 손배 위협 트리아지 한 문단.
  - `ConsultationDoc`에 `damageThreat?: boolean` 추가, 챗봇에서 감지 시 저장 → 어드민 상세에 배지.
- **수용기준**: 손배 키워드("손해배상/위약금/배상") 입력 시 챗봇이 변호사 대응 안내 + 어드민에 플래그 노출.
- **규정**: ✅ 사실 기반 안내. 결과 단정 금지.
- **난이도**: 하 (0.5~1일)

### P0-2. 셀프 진단 → 패키지 자동추천
- **목표**: 챗봇 외에, 3~5문항 구조화 진단으로 **즉시 패키지 + 예상 보수 + 다음 행동**을 제시(이탈 줄이고 결제로 직결).
- **범위**: `/diagnose` 신규 라우트. 문항(고용형태·퇴사사유·미지급 유무·회사 반응·시급도) → 규칙 기반 매핑 → 기본/표준/분쟁 추천 + 계산기 연계.
- **구현**:
  - 신규 `src/pages/DiagnosePage.tsx` + 라우트(`App.tsx`).
  - 매핑 로직은 `chat.ts`의 분류 기준(베이직/표준/분쟁) 재사용.
  - 결과에서 `saveConsultation({source:"form", pickedItems, ...})` + "상담 연결 결제(`/checkout/:id`)"로 유도(=`caseId` 연결 → `/my`·어드민 반영).
- **수용기준**: 진단 완료 시 패키지 1개 추천 + 결제/상담 CTA. `consultations`에 진단 결과 저장.
- **규정**: ✅ 추천은 "일반 정보". 단정 자문 금지 문구 유지.
- **난이도**: 중 (2~3일)

### P0-3. 결제수단 확대
- **목표**: 카드 외 **카카오페이·네이버페이·무이자 할부** 노출로 결제 이탈 감소(일본의 간편결제·後払い 다양화 대응).
- **구현**: `src/pages/CheckoutPage.tsx`의 `tp.requestPayment("카드", …)`를 토스 **결제수단 선택(간편결제 포함)** 으로 확장. 서버 `confirm`은 금액 대조 로직 그대로라 영향 없음.
- **수용기준**: 결제창에 카카오페이/할부 노출, 승인·반영 정상.
- **규정**: ✅ ("무이자 할부"는 카드사 제공이면 가능, "할인" 표현은 금지)
- **난이도**: 하 (0.5일)

### P0-4. 미지급금 성공보수 옵션
- **목표**: 일본 변호사 타입의 成功報酬(회수액 20~30%) 모델을 한국 변호사 보수기준에 맞춰 도입 → **선결제 부담을 낮추고 회수 사안 ARPU↑**.
- **범위**: 표준/분쟁 패키지에 "착수금 + 회수 성공 시 %" 옵션. 결제는 착수금만 즉시, 성공보수는 사후 청구.
- **구현**:
  - `orders`/`consultations`에 `feeType: "fixed" | "contingency"`, `contingencyRate` 필드.
  - 어드민 결제 주문 화면에 성공보수 추적 컬럼.
  - 가격 페이지 카피 + 보수약정 고지.
- **수용기준**: 옵션 선택 시 착수금 결제 + 약정 내용 기록.
- **규정**: ⚠️ **변호사 보수약정·고지 의무** 준수 필요(성공보수 자체는 합법, "환불보장"과 무관). 변호사 본인 검토 필수.
- **난이도**: 중 (정책·약관 포함 3~4일)

## 3. P1 — 변호사 해자 강화

### P1-1. 진행 상황 타임라인 + 카카오 알림톡
- **목표**: 일본의 進捗共有·無期限フォロー. `/my`를 **단계 타임라인**으로 고도화하고, 단계 전환 시 의뢰인에게 **알림톡** 푸시.
- **구현**:
  - `ConsultationDoc.status`(new→contacted→consulted→contracted→closed)를 의뢰인용 단계 라벨로 매핑, `MyPage.tsx`에 타임라인 UI.
  - 단계 변경 시 `functions/api/notify`(SOLAPI) 확장 → **알림톡 템플릿**(승인 필요)로 의뢰인에게 발송. 현재는 변호사에게만 SMS.
- **수용기준**: 어드민이 상태 변경 → 의뢰인 알림톡 수신 + `/my` 타임라인 갱신.
- **규정**: ✅
- **난이도**: 중 (알림톡 템플릿 심사 리드타임 별도)

### P1-2. 증거 보안 업로드
- **목표**: 괴롭힘·분쟁 사안의 녹취·영상·카톡 캡처를 사건에 안전 업로드(변호사 분쟁 대응 차별화). 현재는 챗봇 대화로만 언급됨.
- **구현**: Firebase Storage + `storage.rules`(본인 uid·어드민만), `MyPage`·`ConsultationDetail`에 업로드/열람 UI, `consultations/{id}/files`.
- **수용기준**: 의뢰인 업로드 → 어드민만 열람, 본인 외 접근 차단.
- **규정**: ✅ 비밀유지 의무·개인정보 처리방침 반영.
- **난이도**: 중 (2~3일)

### P1-3. 퇴사 후 서류 + 실업급여 패키지
- **목표**: 일본 離職票 확보·傷病手当 지원처럼, **이직확인서·경력증명서·원천징수영수증 확보 지원 + 실업급여 안내**를 부가 상품/체크리스트로.
- **구현**: 진단/패키지에 "퇴사 후 서류·실업급여 케어" 옵션, 실업급여 자격 간이판정(권고사직 vs 자발 등)을 계산기/진단에 연계. 관련 블로그와 내부링크.
- **수용기준**: 옵션 선택 시 서류 체크리스트 + 실업급여 가이드 제공.
- **규정**: ✅
- **난이도**: 중

### P1-4. 연차 소진 협상 옵션
- **목표**: 有給消化交渉 대응 — 잔여 연차 소진 협상을 패키지 부가 옵션으로 명시.
- **구현**: 패키지 옵션 플래그 + 카피. 분쟁 패키지에 기본 포함 검토.
- **난이도**: 하

## 4. P2 — 성장 베팅

### P2-1. 다국어 / 외국인 근로자 전용
- 한국 외국인력 증가 + 일본 외국인 전용 서비스 존재. EN(필요 시 베트남어 등) 챗봇·진단·랜딩. `usePageMeta` i18n + Claude 프롬프트 다국어화. 기존 블로그 주제(`foreign-worker-resignation`)와 연계.
- **난이도**: 중상

### P2-2. 세그먼트 랜딩
- 직장 내 괴롭힘 전용 / 5인 미만 전용 랜딩(SEO·전환). 일본 여성전용(わたしNEXT) 사례. **난이도**: 하

### P2-3. 실적 카운터 · 후기 강화
- 정량 실적·후기(규정 준수 범위, 최상급 금지). 후기 어드민 이미 존재 → 노출 강화. **난이도**: 하

## 5. 권장 시퀀스 & KPI

**시퀀스**: P0-1(손배 방어) → P0-3(결제수단) → P0-2(셀프 진단) → P1-1(타임라인·알림톡) → P0-4(성공보수) → P1-2(증거 업로드) → 이후 P1·P2.

**KPI**:
- 방문 → 상담신청 전환율
- 상담신청 → 결제 전환율 (P0 핵심)
- 패키지 믹스(표준·분쟁 비중 = 해자 상품 판매)
- ARPU(성공보수 도입 후)
- 사건당 완료 리드타임(타임라인 데이터 기반)
